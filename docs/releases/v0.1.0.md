# sensorium v0.1.0

## Highlights

- first public release of the `sensorium` platform
- generic family/profile driver architecture with Sony IMX as the first backend
- 61 selectable IMX sensor profiles
- userspace-fed injection plus libcamera-detectable capture
- local and remote validation workflows, including full-catalog sweeps
- Debian, Alpine, and Arch DKMS packaging scaffolding with GitHub Actions builds

## Included validation work

- raw cadence validation at `10`, `20`, and `30 fps`
- raw and processed smoke capture
- MP4-backed inject and record workflows
- full catalog sweep with targeted clean rechecks for initially failing profiles

## Known limitations

- processed/viewfinder throughput is still host-ISP-dependent
- the project targets camera-software-compatible simulation, not full physical
  reproduction of each Sony sensor
- current family support is IMX only, though the architecture is generic
